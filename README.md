# ATL Birth Hub

A polished Streamlit web app helping expecting parents in Metro Atlanta (~60-mile radius) compare birthing facilities with calm, supportive guidance.

![Python](https://img.shields.io/badge/Python-3.10%2B-blue)
![Streamlit](https://img.shields.io/badge/Streamlit-1.32%2B-red)

## Features

- **Hero landing** with warm, empowering copy and sage/teal + terracotta design
- **ZIP-based distance filter** with priority-based facility matching
- **Compare Options** — searchable, sortable comparison table
- **Facility Profiles** — rich cards with costs, quality, strengths, and considerations
- **Map & Nearby** — Streamlit map preview (Folium-ready placeholder)
- **Resources & Support** — curated local prenatal, lactation, mental health, and community resources
- **Clear disclaimers** — costs are estimates; not medical advice

## Data Coverage

The app loads **45+ facilities** from CMS Hospital Compare:

- All Georgia **acute-care** and **critical-access** hospitals within **60 miles** of downtown Atlanta
- Excludes psychiatric, children's-only, and VA hospitals
- Plus accredited **birth centers** (e.g., Atlanta Birth Center)
- Hand-curated detail for major hospitals; CMS-derived estimates for the rest

Refresh from CMS anytime:

```bash
python data_ingestion.py --refresh
```

## Quick Start

### 1. Clone or navigate to the project

```bash
cd atl-birth-hub
```

### 2. Create a virtual environment (recommended)

```bash
python3 -m venv .venv
source .venv/bin/activate   # macOS/Linux
# .venv\Scripts\activate    # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. Refresh facility data from CMS (recommended)

```bash
python data_ingestion.py --refresh
```

This fetches Georgia hospitals from CMS, filters to Metro Atlanta, and writes `data/facilities.csv`.

### 5. Run the app

```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## Customizing Data

### Option A: Edit the CSV

Run `python data_ingestion.py` to generate a starter CSV, then edit `data/facilities.csv`. Required columns:

- `facility_id`, `name`, `type`, `address`, `zip_code`
- `latitude`, `longitude`
- `vaginal_cost`, `csection_cost`
- `quality_rating`, `cms_stars`, `csection_rate`
- `nicu_level`, `birth_center`, `teaching_hospital`
- `strengths`, `considerations`
- `priorities` (pipe-separated, e.g. `Lower costs|Close to home`)

### Option B: Extend `data_ingestion.py`

See the MRF integration comments in `load_facilities()` for parsing hospital Machine-Readable Files and merging CMS quality data.

## Disclaimers

Cost figures are illustrative planning estimates. Actual charges vary by insurance, complications, and individual care. Quality metrics are summarized from public sources. Always consult your healthcare provider and hospital billing office.

## Deployment

### Streamlit Community Cloud (recommended)

The repo is pre-configured for [share.streamlit.io](https://share.streamlit.io).

**1. Push to GitHub**

```bash
cd atl-birth-hub
# Create a new empty repo at github.com/new named "atl-birth-hub"
git remote add origin https://github.com/YOUR_USERNAME/atl-birth-hub.git
git push -u origin main
```

**2. Deploy on Streamlit Cloud**

1. Sign in at [share.streamlit.io](https://share.streamlit.io) with GitHub
2. Click **Create app** → **Yup, I have an app**
3. Set:
   - **Repository:** `YOUR_USERNAME/atl-birth-hub`
   - **Branch:** `main`
   - **Main file path:** `app.py`
4. Click **Deploy** — first build takes ~2–3 minutes

Your public URL will look like: `https://atl-birth-hub.streamlit.app`

**3. Keep data fresh (optional)**

A GitHub Action (`.github/workflows/refresh-data.yml`) can open weekly PRs with updated `data/facilities.csv`. Enable it by pushing to GitHub — no Streamlit secrets required.

### Option B — Docker (Railway, Render, Fly.io, AWS, GCP)

```bash
docker build -t atl-birth-hub .
docker run -p 8501:8501 atl-birth-hub
```

Railway / Render: connect the repo, use the Dockerfile, expose port **8501**.

### Option C — Keep it local / private beta

Run on your machine or LAN only. No deployment needed until you want a public URL.

## Security

**Good news:** ATL Birth Hub is a **read-only public information tool**. Users do not log in, submit forms, or enter health data. That keeps the security bar much lower than a patient portal.

| Concern | Needed for this app? | Notes |
|--------|----------------------|-------|
| User authentication | **No** (public MVP) | Add later only for admin/analytics dashboards |
| HTTPS | **Yes** (production) | Provided automatically on Streamlit Cloud, Railway, Render |
| API secrets | **No** today | CMS data is public; use `.streamlit/secrets.toml` if you add paid APIs |
| HIPAA | **No** | No PHI collected — don't add name/DOB/insurance fields without legal review |
| Rate limiting | **Optional** | Useful if traffic spikes; platform CDNs often help |
| Input validation | **Light** | ZIP is 5 digits — already validated |
| Dependency updates | **Yes** | Pin versions in `requirements.txt`; update quarterly |
| Disclaimers | **Yes** | Already in app — keep visible (not medical/financial advice) |

### Recommended before going public

1. **Commit `data/facilities.csv`** so production does not depend on live CMS calls at startup.
2. **Never commit** `.streamlit/secrets.toml` (listed in `.gitignore`).
3. **Turn off** Streamlit usage stats in production (already set in Docker / config).
4. **Optional password gate** for a private preview — only if you want to limit access before launch; not required for a public parent-facing tool.
5. **Legal review** if you add user accounts, insurance IDs, or personalized cost quotes tied to identity.

### When you *would* need stronger security

- Storing user profiles or due dates
- Connecting to insurer APIs with member IDs
- Letting hospitals edit their own listings (admin roles, auth, audit logs)
- Processing payments or bookings

None of those apply to the current MVP.
